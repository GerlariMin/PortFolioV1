CEDR
Présentation Fonctionnalités Langages/ Outils Galerie Ressources

La mission principale de mon stage était de créer un site Web interne permmettant d'automatiser et de simplifier les procédures CEDR (Commission d’Évaluation des Demandes de Recrutement) au sein de la mairie d'Aubervilliers.

Le site web devait également permettre d'avoir une structure bien précise pour pouvoir poser les bases d'une architecture de site MVC (Modèle Vue Contrôleur) et Responsive Design qui pourraient être reprisent pour de futurs projets.

Une personne ou un service qui veut effectuer une demande de recrutement ou de remplacement devra donc accéder à cette plateforme pour compléter le formulaire adapté et pouvoir le transmettre au service qui s'occupera de traiter cette demande.

Un demandeur doit pouvoir:

  • Accéder au site avec son adresse mail Zimbra et un code personnalisé
  • Consulter l'état de sa demande
  • Déposer une demande à une séance non clôturée
  • Modifier sa demande
  • Remplir l'un des formulaires correspondant à sa demande
  • Supprimer sa demande
  • Voir les séances de CEDR qui ne sont pas encore clôturées

Un gestionnaire doit pouvoir:

  • Avoir les mêmes fonctionnalités que le demandeur classique
  • Gérer les demandes (ajout, modification, suppression)
  • Gérer les séances CEDR
  • Gérer les tables des différents services, grades et secteurs
  • Saisir les avis des Ressources Humaines et les décisions des commissions (favorables ou non)

Le site doit:

  • Être Responsive Design
  • Avoir une structure MVC
  • Permettre l’extraction des demandes afin de disposer d’un document Excel contenant l’essentiel des informations nécessaires aux directions

  • AdminBSB
  • BootStrap
  • CSS
  • HTML
  • MySQL
  • MySQL Workbenc
  • PHP
  • Talend Open Studio
  • WampServer

LoutreBay
Présentation Fonctionnalités Langages/ Outils Galerie Ressources

Le site LoutreBay est un site de vente de loutres développé dans le cadre de Travaux Pratiques et d'un projet individuel.

En effet, un visiteur peut avoir accès à la liste des ventes des loutres, aux détails des informations d'une loutre précise et peut mettre en vente une loutre s'il est connecté.

L'objectif de ce site est de mettre en place un site Responsive Design utilisant des méthodes de bonnes pratiques en développant de manière autonome

Un utilisateur doit pouvoir:

  • Accéder aux informations détaillées d'une loutre en particulier en ayant cliqué dessus
  • Consulter ses ventes (ou toutes s'il est administrateur)
  • Mettre en vente une loutre (via un formulaire)
  • Modifier ses ventes (ou toutes s'il est administrateur)
  • Se connecter à son compte si il est déjà inscris
  • Se créer un compte (via un formulaire)
  • Supprimer ses ventes (ou toutes s'il est administrateur)
  • Trier la liste par ordre de prix (croissant ou décroissant)
  • Trier la liste par rapport à la race (ordre alphabétique)
  • Voir la liste des loutres en ventes

Le site doit:

  • Assurer le sécurité du mot de passe de l'ustilisateur en le cryptant
  • Attribuer les bons droits en fonction du type de compte (administrateur ou non)
  • Avertir l'utilisateur si le pseudo saisi lors de l'inscription est déjà utilisé
  • Être Responsive Design

  • BootStrap
  • CSS3
  • Font Awesome
  • GitLab
  • HTML5
  • JavaScript
  • MySQL
  • MustacheJS
  • PHP

Ludéma
Présentation Fonctionnalités Langages/ Outils Galerie Ressources

Le site est issu de celui déjà existant de la Stat-Up Ludéma .

Le but de ce site est de permettre à toute personne (quelque soit sa mobilité) de pouvoir avoir accès au sport et à de l'équipement, que ce soit en salle ou non.

Dans le cadre de mon projet de groupe, la Start-Up nous a demandé de rajouter des fonctionnalités afin de permettre à un visiteur de s'inscrire et de participer à une séance.

Un utilisateur doit pouvoir:

  • Accéder à ses données
  • Consulter le suivi des bilans fournis par un professionnel
  • Consulter l’agenda des séances
  • Indiquer ses préférences pour les séances (lieux, matériel)
  • Modifier ses données
  • Se connecter avec un compte déjà existant si il est déjà membre
  • Supprimer ses informations ou son compte si il ne souhaite pas continuer d’utiliser le service
  • S'inscrire si il est un nouveau membre

Le site doit:

  • Assurer une charte graphique simple et adaptée pour un plus large public

MonkeyBall
Présentation Fonctionnalités Langages/ Outils Galerie Ressources

Le projet MonkeyBall est une application Web développée pour les appareils Android.

L'objectif est de pouvoir jouer au jeu en inclinant l'appareil pour pouvoir déplacer la balle.

Le but du jeu est de faire rentrer la balle dans l'un des 8 trous, sachant qu'il n'y a qu'un seul trou gagnant (en vert) et les 7 autres sont perdants (les noirs).

Le projet MonkeyBall a été développé en HTML/ CSS et JavaScript. Il a été ensuite transformer en APK avec l'outil Cordova.

L'ajout d'une interface 3D est envisagée pour la finalisation du jeu.

L'application doit:

  • Être intuitive
  • Être Responsive Design pour une navigation sur tout type d'appareils
  • Gérer les interactions Homme/Machine
  • Gérer les rebonds de la balles
  • Générer des trous de manière aléatoire sur le plateau
  • Utiliser le gyroscope de l'appareil pour déplacer la boule

  • BootStrap
  • CSS
  • Font Awesome
  • HTML
  • Cordova
  • Android (Pour tester l'application)
